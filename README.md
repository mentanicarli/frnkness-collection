# frnk ness collection

Музыкальный веб-проект с коллекцией релизов, плеером, текстами песен и чартом прослушиваний.

Сайт сделан для личного применения небольшой группой людей.
Исполнитель песен на сайте: frnk ness.

## Текущее состояние

Проект мигрирован с монолитного HTML в современную структуру на Vue + Vite.

Также настроены:

- PWA-поддержка (manifest, иконки, добавление на главный экран)
- iOS Home Screen мета-теги для Safari
- Фавиконки для браузеров (favicon.ico, 16x16, 32x32)

### Технологии

- Vue 3
- Vite 4
- Tailwind CSS (через CDN)
- Supabase JS (через CDN, для счетчика прослушиваний)
- Color Thief (через CDN, для динамических акцентов по обложке)
- Web App Manifest (PWA)

## Возможности

- Главная страница с альбомами и синглами
- Страница релиза с треклистом
- Мини-плеер внизу экрана
- Полноэкранный плеер
- Отображение текстов песен
- Поддержка txt-текстов
- Поддержка lrc с подсветкой активной строки (караоке)
- Чарт треков по количеству прослушиваний из Supabase
- Динамическая цветовая тема от обложки релиза
- Анимации и адаптивный интерфейс
- Установка на iPhone через Safari как веб-приложение с иконкой
- Отдельная настройка favicon для вкладки браузера

## Адаптивность

Проект адаптирован для:

- ПК
- Планшетов
- Телефонов (включая safe-area и динамическую высоту экрана)

## Структура проекта

- index.html — входная HTML-страница
- src/App.vue — основная разметка приложения
- src/legacy/app-core.js — логика приложения (плеер, навигация, тексты, чарт)
- src/assets/app.css — стили
- public/manifest.webmanifest — манифест веб-приложения
- public/apple-touch-icon.png — иконка для iOS Home Screen
- public/pwa-icon-192.png — PWA-иконка 192x192
- public/pwa-icon-512.png — PWA-иконка 512x512
- public/favicon.ico — favicon для браузеров
- public/favicon-16x16.png — favicon 16x16
- public/favicon-32x32.png — favicon 32x32
- audio/ — аудиофайлы
- images/ — обложки и изображения
- lyrics/ — тексты песен
- lyrics-books/ — PDF-сборники текстов
- vite.config.js — конфигурация Vite
- .github/workflows/deploy-pages.yml — автодеплой на GitHub Pages

## Локальный запуск

Требования:

- Node.js 18+ (рекомендуется 20)

Команды:

```bash
npm install
npm run dev
```

## Сборка

```bash
npm run build
npm run preview
```

Готовая сборка появляется в папке dist.

## PWA и иконки

Для iPhone (Safari -> На экран Домой) используются:

- public/apple-touch-icon.png
- public/manifest.webmanifest

Для вкладок браузера используются:

- public/favicon.ico
- public/favicon-16x16.png
- public/favicon-32x32.png

Если меняете логотип, сохраняйте размеры и имена файлов либо обновляйте ссылки в index.html и manifest.webmanifest.

## Деплой на GitHub Pages

В проект уже добавлен workflow для автодеплоя.

Что нужно сделать в репозитории:

1. Запушить проект в ветку main (или master)
2. Открыть Settings -> Pages
3. В Source выбрать GitHub Actions
4. Дождаться завершения workflow Deploy to GitHub Pages

## Важные замечания

- Для работы чарта нужна настроенная база Supabase и функция increment_play_count
- Ключи и URL Supabase сейчас подключены в клиентской части
- При необходимости можно вынести конфигурацию в переменные окружения

## Назначение проекта

Проект ориентирован на личное использование небольшой группой людей, а не на массовый публичный сервис.
